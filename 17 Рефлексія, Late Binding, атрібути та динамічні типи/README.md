# Рефлексія, Late Binding, атрібути та динамічні типи

Як було показано, збірки є основною одиницею розгортання у всесвіті .NET. Використовуючи інтегрований Object Browser в Visual Studio (та багатьох інших IDE), ви можете досліджувати типи в наборі збірок, на які посилається проект. Крім того, зовнішні інструменти, такі як ildasm.exe, дозволяють вам зазирнути в базовий CIL-код, метадані типів та маніфест збірки для заданого бінарного файлу .NET. Окрім цього дослідження збірок .NET під час розробки, ви також можете програмно отримати цю ж інформацію, використовуючи простір імен System.Reflection. З цією метою першим завданням цього розділу є визначення ролі рефлексії та необхідності метаданих .NET.
У наступних розділах цієї глави розглядається кілька тісно пов'язаних тем, що стосуються служб рефлексії. Наприклад, ви дізнаєтесь, як клієнт .NET може використовувати динамічне завантаження та пізнє зв'язування для активації типів, про які він не знає під час компіляції. Ви також дізнаєтеся, як вставляти власні метадані у ваші збірки .NET, використовуючи системні та власні атрибути. Щоб розглянути всі ці (здавалося б, езотеричні) теми в перспективі, розділ завершується демонстрацією того, як створити кілька «об’єктів оснащення», які можна підключити до розширюваної консольної програми.
У цьому розділі ви також ознайомитеся з ключовим словом dynamic у C# та зрозумієте, як вільно типізовані виклики відображаються на правильний об'єкт у пам'яті за допомогою Dynamic Language Runtime(DLR). Після того, як ви зрозумієте служби, що надаються DLR, ви побачите приклади використання динамічних типів для оптимізації виконання викликів методів із запізненням (через служби відображення) та для легкої взаємодії зі застарілими бібліотеками COM.
